Memory Category Commands

This section is basically the most complicated set of commands on the bot, especially tags.
Most of the commands in this section are based around storing and recalling data.

=event

"lets you access my calendar functions"

This command allows users to recall upcoming events.
These are either events set by the mods or birthdays set in user's profiles.

Usage:
=event [list]
Lists all birthdays and events in the server. If there are too many, a file will be posted.
=event next
Lists only birthdays and events in the next month. Useful if there are too many on the full list.

=event [Bot Mod]

This command allows you to set up events that can be seen by members on the server.
However, there wont be any alerts when the event hits and it will not be removed automatically after it does.

You cannot manage birthdays using this command. Birthdays will always be present on the list.

Usage:
=event create 'date' 'message'
Adds an event to the list. Date is the date to add to, message is the message to add.
=event set 'date' 'message'
Deletes all events on a date and replaces it with the given event.
=event delete 'date'
Deletes an event from a date. If there is more than 1, this is interractive.
=event clear 'date'
Deletes all events from a date.
=event wipe
Deletes all events on the list.

=profile

"lets me keep track of your online profile"

This command lets you tell 42 information about yourself to allow other users to easily access this information.
Having a profile is entirely optional.

Beta 42 respects the visibility principle, which is an unwritten rule in Discord that means that if a user cannot see a server, channel, or user, then that server, channel or user does not exist.

Some elements of profiles are used in other commands, such as timezones in =now, birthdays in =event and both in =solstice.

Usage:
=profile [get ['member]]
Gets a profile. Member is who's profile to get, or if no name is given, then it gets your own.
=profile nnid 'data'
Sets your default Nintendo Network ID. Data is the new NNID
=profile alt 'data'
Adds an alternate Nintendo Network ID. Data is the NNID to add. You can have up to 3 alts.
=profile game 'data'
Adds a new game to your profile. Data is the game to add. You can have up to 8 games.
=profile birthday 'date'
Adds a birthday to your profile. Date is your birthday.
=profile timezone
Sets your timezone. This command is interractive and can only be done in PMs.
=profile note 'data'
Sets your profile note. Data is the new note. Notes are displayed at the bottom of your profile.
=profile set 'field' 'data'
Sets a custom field. Field is the field name, data is the data to set. You can have up to 5 fields.
=profile update
Updates a field. This command is interractive.
=profile clear 'field'
Clears a default field. field is one of "nnid", "game", "birthday", "timezone", "note"
For games and NNIDs, the command is interractive.
=profile clear custom
Clears a custom field. This command is interractive.
=profile delete
Deletes your entire profile.
=profile make
Deletes your existing profile and starts an interractive walkthrough to create a new one.
This command can only be used in PMs.
You can use =skip to skip any stage of the process.

=tag [Old User] [5/15s]

"lets me store and recall messages, image links or custom scripts"

Permission: [Old User] Caller must either have been a member of the server for 15 minutes or have a role with more permissions than @everyone.

Tags are like recallable bookmarks.
You can store long messages or image links under a easy to remember name, and then recall the images later.

Tags can contain scripting using a custom language.

Internal help docs are spread over 4 different help commands, =help tag, =help taginfo, =help tagmovement and =help tagyna

Tags are saved exclusively per server. Currently there is no way to

Usage:

For all subcommands, name is the name of the tag you want to interract with.

<user> =tag create foo "Hello World"
<B 42> I have created your 'foo' tag.
<user> =tag foo
<B 42> Hello World
<user> =tag delete foo
<B42> I have deleted that tag.


Basic Tag Editing
=tag 'name'
Recalls a tag with the given name.
=tag create 'name' 'content' ['init']
Creates a new tag. Content is the content to save.
You cannot create a tag with a name that is already taken or reserved.
You will be assigned as the owner of this tag.

Init is the code that will be run when the tag is created. It is intended to be used to set up state variables.
=tag delete 'name'
Deletes a tag.
You must own the tag or be a Bot Mod to delete tags.
=tag edit 'name' 'new content'
Edits an existing tag. New content is the content to save.
You must own the tag or be a Bot Mod on the server.
If the tag is synced, then you must be the owner to edit it.
=tag extend 'name' 'line'
Appends content to the end of a tag starting on a new line. Line is the content to append.
Tags marked as public can be extended by anyone allowing for the creation of request lists.
=tag rename 'name' 'new name'
Renames an existing tag. New name is the new name for the tag.
The new name cannot be a name that is already taken or reserved.
You must own the tag or be a Bot Mod to rename tags.
If the tag is synced, then you must be the owner to edit it.
=tag gift 'name' 'member'
Transfers ownership of a tag to another user. Member is who you want to give the tag to.
Gifting a tag will remove you as the owner, meaning you can no longer edit or delete it.
You must currently own the tag to gift it.

Other Basic Functions
=tag list ['member']
Gets a members tag list. Member is who's tags to get, if no member is given, it gets your own.
Only tags for servers you are on will be shown. If you do not share a server with your target, you will not see those tags.
If you have orphaned tags on a server, those will not be shown either.
=tag list all Posts a (paginated) list of every tag in the server. This list also contains who created the tag and what metadata flags are on the tag.
  • [HDN] This tag is hidden: Calls will be deleted.
  • [YNA] This tag contains yna scripting.
  • [CLS] This tag is closed source: You cannot call it with raw.
  • [NSW] This tag is NSFW: It cannot be used outside NSFW channels.
  • [SNC] This tag is synced.
  • [NMV] This tag is restricted: It cannot be cloned, pulled or exported.
  • [SPL] This tag contains spoilers: It will not appear in random.
=tag search 'query'
Searches for a tag by name. Query is the substring to search for.
=tag random
Posts a random tag from the server.
Tags marked spoiler or NSFW will not be pulled in random.
=tag info 'name'
Fetches info about a tag, who owns it and what metadata flags it has.
=tag claim 'name'
Gives yourself ownership of an orphaned tag.
Tags becomed orphaned when their owner has left the server.

Tag Metadata

Every tag has a set of metadata flags that can be enabled and effect how the tag works.

=tag spoiler 'name'
Marks a tag as spoiler content.
  • Spoiler tags will not appear in =tag random
  • Spoiler tags will be locked to spoiler rooms if your server has defined them.
=tag restrict 'name'
Prevents this tag from being copied to/from other servers by members who are not the owner of the tag.
=tag hidden 'name'
Makes the bot attempt to delete calls to this tag.
The bot requires Manage Messages to do this.
=tag nsfw 'name'
Marks the tag as NSFW content.
  • NSFW tags can only be used in discord marked NSFW channels.
  • NSFW tags will not appear in =tag random
=tag closed 'name'
Marks a tag as closed source. Tags with this cannot be called with =tag raw unless the caller is the tag's owner.
=tag public 'name'
Marks a tag for public editing.
Tags with this can be extended with =tag extend by any member of the server.
Edits that would otherwise clash the tag with one of the owner's synced tags cannot be made.
=tag sync 'name'
Syncronises a tag with any identical tags accross servers.
The check for matching tags is same owner and content, not name.
Any edits to the content or metadata of one synced tag will propogate through the rest of the connected tags.

Cross Server Tag Management

Tags are strictly per server. This method can be used to transfer tags between servers.
A note on clone and pull:

  • You cannot clone to or pull from servers where tags are disabled via toggle.
  • You cannot clone to or pull from servers that you are not a member of.
  • You cannot clone or pull tags which are restricted unless you are the tag's owner.

=tag clone 'name' 'server' ['new name'] Copies a tag to a given server. Server is the target server, new name is the name the tag will assume on the target server.
=tag pull 'name' 'server' ['new name'] Copies a tag from a given server. Server is the target server, new name is the name the tag will assume on the current server.
=tag export Dumps all of your tags into a file, which is uploaded to chat.
Restricted tags will not be copied into the file.
=tag import This command only works when it is the caption of a file upload.
The file must be one generated by the =tag export command.
If your file contains NSFW tags, you will be prompted to import them seperately.

Scripting

Tags can use a custom scripting language called Yonni Na.
The tools defined below can help with debugging your scripts.

=tag raw 'name'
Fetches the raw content of a tag.
If the tag is closed source, this will not work.
=tag eval 'content'
Evaluates arbitrary yna objects. Content is the content to evaluate.
=tag debug 'name'
Executes a tag and then posts a debug log.
=tag deval 'content'
A combination of debug and eval. Content is the content to evaluate.

Extra Functions

These functions aren't for interracting with specific tags.

=tag spoilerrooms '#channel' [...] Sets the servers spoiler rooms. There is no way to individually add rooms, so you must define all at once.
#channel is the channel mention. You can have multiple mentions.
If a spoiler room is defined, then tags marked as spoilers will not be usable outside them.
=tag spoilerrooms clear Removes the spoiler room restriction from the server.
=tag welcome ['new content'] Sets the welcome message for the server. New content is the content to set for the welcome message, if none is given, then the current message is returned.
{user} and {@user} can be used to name or mention the new user.
