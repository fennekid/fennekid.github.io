Help for 42

"Hello! I am 42 and I am a robot. I wish to make your time inside Discord easier and more enjoyable. On this site, there is a full description of all my commands and how to use them. I look forward to seeing you over on your server~~!"

Commands:
All commands are called with a + sign, but an ! can also be used. If @R. Danny is on the server, common commands such as +schedule and +info will not work with an ! and must be called with a +

Server and Bugs: Found a bug? have a suggestion? need help? bored and want someone to talk to? I have a server you can check out. There is an invite in the +about message or if you prefer, just look to the right and there is an invite there.
Also, to clarify, all paths 42 can take whether successful or erroring should in 99% of cases send something back to chat so if this doesnt happen, you can consider it a bug and let me know what you did over in the Kiwi Hangout

Authority:
There are several authority levels that 42 uses. Any [Server] level authorities are given by having certain permissions

  • [Server Owner] This is the big man, woman or Wumpus in charge of the server.
  • [Server Admin] These are the people who have just as much power. They have "Manage Roles"
  • [Server Manager] These are the second in command. They are the ones with "Manage Servers"
  • [Server Mod] These are the guys who keep the peace. They have "Manage Messages"
  • [Kick] and [Ban] These lot are the ones to watch out for since they have "Kick" and "Ban"
There are a couple of authority levels that mods can hand out without having to give away power on the server. They are [Bot] roles
  • [Bot Admin] These people keep the bot under control since she sees them as "Best Friends" (⇐ role name btw)
  • [Bot Mod] These people keep chat free of the bots mess since they are "Friends" (⇐ also role name)
There are a couple special ones too
  • [Old User] These people who werent banned in the first 15 minutes for spamming become old users
  • [User] This is a group for everyone. Any command that isnt marked with another auth level is a [User] command
  • [Buds] This is the exclusive group of people squas hangs out with. Don't ask to join, he will say no.

Category Moderation

+remove [Server Mod] [User]

This command makes 42 clear a specific set of messages out of the channel.
42 requires the "Manage Messages" permission to use this function.

Basic Usage: User commands

  • +remove me ['limit'] archives chat and removes all messages from the user.
Server Mod commands
  • +remove user 'username' ['limit'] removes all messages from the given user
  • +remove user @user [@user ...] removes entire conversations
  • +remove embeds ['limit'] removes all embeded messages from chat
  • +remove contains 'query' ['limit'] removes all messages that contains the given query
  • +remove commands 'calls' ['limit'] removes messages that start with specific command calls
  • +remove all ['limit'] removes all messages from the chatlog

Notes: limit is how many messages are searched, not how many are removed and in +remove commands, the calls are a space seperated list, for example '+! ~ `'

+welcome [Server Manager]

This makes 42 start (or stop) listening for new users joining the server and announces their arrival.

Basic Usage:

  • +welcome 'code'

+welcome has (currently) 7 different modes.

  • 1: Post standard message
  • 2: Post welcome hug
  • 3: PM server owner
  • 4: Post the contents of +tag welcome
  • 5: Wait until they are about to post
  • 6: Wait and then post +tag welcome
  • 0: Do nothing

Note: options 5 and 6 are reliant on 42's uptime. If she goes down, the buffer of new users is dropped.

+blacklist [Server Mod]

This makes 42 listen out for specific words in non moderator messages and if that word appears, she will delete the message.
42 requires the "Manage Messages" permission to use this function.

Basic Usage:

  • +blacklist add 'term' adds a word to the filter
  • +blacklist remove > 'number' removes a term from the filter
  • +blacklist list lists the words in the filter
  • +blacklist clear clears the filter

Note: 42 requires constant access to manage messages.

+giveme [Server Admin] [User]

This lets 42 remember and hand out certain roles to users without requiring continued admin input
42 requires the "Manage Roles" permission to use this command.

Basic Usage: Server Admin commands

  • +giveme set 'name' 'rolename' adds a giveme to the list
  • +giveme clear 'name' removes a giveme
  • +giveme exclusive marks (or unmarks) the servers givemes as mutually exclusive
User commands
  • +giveme list lists the givemes on the server
  • +giveme 'name' makes 42 give the caller the role associated with name

I'm sure that you can come up with more creative ways to use this, but so far I've seen servers use givemes for access to NSFW or spoiler channels and also use mutually exclusive givemes to let users choose their colour.

Example Usage

>>> +giveme set 1 "Custom Colour 1"
42: I have set `+giveme 1` to assign custom colour 1
>>> +giveme 1
42: I have given you `1`

+brainfuck

Brainfuck is an esoteric programing language that consists of only 8 characters. if you give 42 some brainfuck code, she will compile and execute it and return the results.

Basic Usage:

  • +brainfuck 'code' ['input']

Quick reference:

  • > moves the pointer (^) 1 cell right
  • < moves the pointer (^) 1 cell left
  • + increments the value at ^
  • - decrements the value at ^
  • [ denotes an open loop and will skip to the corresponding ] if the value at ^ is 0
  • ] denotes a close loop and will jump back to the corresponding [
  • . prints the ascii character at ^
  • , reads the next input character into ^

+tag

This is 42's database function. It allows users to store and recall messages and image links.

  • +tag create 'name' 'contents' and
  • +tag create 'name' > 'contents' allow a user to create a tag
  • +tag rename 'name' 'new name' renames a tag
  • +tag edit 'name' 'new contents' modifies a tag
  • +tag clone 'tagname' 'server name' copies any tag to another server and makes you the owner of that copy

  • +tag search 'query' will search the database
  • +tag clain 'tagname' makes you the owner of a tag if the original owner has left

Tags are exclusively per server. Commands such as clone, export and import can be used to copy tags between servers.

The +uplink command

Now then. I feel like the existance of this command requires justification. Most servers never need to worry about it though as I will hopefully never need to use it.
First, a little bit of context. Even though 42 is a bot, she is still affected by certain limitations within Discord. The most problematic of this is the message rate limit. In the past when I have seen 42's log fill up with a large amount of identical requests (such as tag spam) I have needed to log into her account to access the offending server and let them know that the rate limiting she is experiencing is ruining it for everyone.
This was before the official API came out however. Now we have bot accounts which are slightly more leniant on the restrictions but at the cost of unlinking her email and password meaning I will have no way to access the bot anymore to contact these problem servers.
This is where +uplink comes in. It is a command that only I (squaswin) can use and it connects 2 channels on different servers and make the bot relay messages between them. I hope I never need to use this command, but if I do, you will know when I do. 42 will alert you when she starts and stops a connection.

42: This server has been uplinked to Kiwi Hangout#ishinomoris-lab. Any messages sent in either channel will be relayed between them.
42: squaswin: yes, hello, can you please stop spamming the bot. There are rate limits and you are breaking her.
42: The connection has been closed. No more messages will be relayed.
